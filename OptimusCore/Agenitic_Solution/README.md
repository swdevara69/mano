# TestCase_CustomAgents

This repository is designed to support the development and use of custom agents for generating user stories and testcases using contextual information.

## How to Use

1. **Context Agent** 🧠
   - The context agent helps you generate a detailed user story.
   - Provide input such as a Jira content Markdown file or a screenshot with a short description.
   - You can also include reference documents like navigation steps or a template file.
   - The agent curates a comprehensive user story which becomes the new input for the testcase agent.

2. **Testcase Agent** ✅
   - Use the generated user story as input for the testcase agent.
   - The agent will create testcases in your desired format. You may supply a template if needed.

3. **Base Repository Structure** 📁
   - All your input documents should be stored in the `reference` folder.
   - Generated user stories and template files are also stored here.

4. **Prompt Configuration** ⚙️
   - The `3_Prompt_Config/` directory contains prompts for the custom agents.
   - There are both short and detailed prompts depending on your usage.

5. **Design Studio** 🎨
   - The `4_Design_Studio/` directory holds the final output files generated by the agents.

6. **Scripts** 🧩
   - Do **not** modify the files in the `scripts/` folder.
   - These Python scripts support context engineering functionality and are essential for the agents to work properly.

Feel free to explore each area of the repository and follow the steps above to generate user stories and testcases efficiently.
